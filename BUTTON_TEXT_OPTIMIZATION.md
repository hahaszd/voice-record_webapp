## 🎯 为什么这样改？

### 用户心理分析

#### 原来："停止录音"
```
用户看到：停止录音
用户想法：❓ 停止了然后呢？会自动转录吗？
焦点：停止 ← 负面动作
```

#### 现在："转录"
```
用户看到：转录
用户想法：✅ 对！我就是要转录！
焦点：转录 ← 用户的真实目标
```

---

## 💡 设计理念

### 目标导向设计（Goal-Oriented Design）

```
用户真正想要的是什么？
    ↓
【转录】音频内容
    ↓
而不是【停止】录音

所以按钮应该说"转录"，而不是"停止"
```

### 认知负担减少

```
改进前：
用户心智模型：录音 → 停止 → ？ → 转录
（需要推理）

改进后：
用户心智模型：录音 → 转录
（直接明了）
```

## 🎓 设计原则总结

### 学到的经验
```
1. 按钮文字应该反映用户目标，而不是系统动作
2. "转录"比"停止录音"更直接
3. 简洁有力的文字降低认知负担
4. 目标导向设计提升用户体验
```

### 可应用的场景
```
✅ 下载按钮：不要写"准备下载"，直接"下载"
✅ 提交按钮：不要写"验证并提交"，直接"提交"
✅ 支付按钮：不要写"确认支付"，直接"支付"

原则：用户想要什么，按钮就写什么
```
